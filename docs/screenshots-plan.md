# Play Store screenshots — shot list (Journal v1)

Play requires **2–8 phone screenshots** (min 320 px, 16:9 or 9:16, PNG/JPEG).
Capture on the SM A515F after a build with the latest wording installed
(the "Back up" menu label landed 2026-05-31 — screenshot the new build, not
an older one).

## Prep

- Seed a few realistic-looking entries across different dates so search and
  Past entries don't look empty. Avoid anything private — these are public.
- Light theme, full battery / clean status bar if possible.
- Same device/aspect for all shots so the set looks consistent.

## Shots (in store order — first 2 matter most, they show in search results)

1. **The hook: blinking cursor on today's page.** The empty/just-started
   today page with the cursor and date. Caption idea: "Two taps from locked
   to writing."
2. **Fingerprint unlock screen** (auth_gate). The single fingerprint tap zone.
   Caption: "Open. Touch. Write."
3. **An entry being written**, with session timestamps visible. Caption:
   "Auto-saves every word — nothing to press."
4. **Full-text search** results with a highlighted snippet. Caption:
   "Find anything you've ever written."
5. **Past entries** list by date. Caption: "Your whole journal, by date."
6. **Overflow menu open** showing Search / Past entries / Back up. Caption:
   "Back up to plain Markdown — your data, no lock-in." (reinforces the
   privacy + backup story)

Shots 1–4 are the strong core; 5–6 are optional if we want to fill the set.

## After capture

- Optional: frame in a device mockup + add the caption text band (tools like
  shotbot / screenshots.pro / a simple Figma template). Plain raw screenshots
  are acceptable for v1 if time is short.
- Drop final files somewhere out of the build dir before uploading.
